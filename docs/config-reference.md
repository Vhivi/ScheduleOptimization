# Configuration Reference (`backend/config.json`)

This guide explains each field in `backend/config.json`, expected types, and common pitfalls.

For machine validation, use:
- Schema: `backend/config.schema.json`
- Example: `backend/config.example.json`

## Local File Policy

- `backend/config.json` is a local runtime file and is ignored by Git.
- To initialize it, copy `backend/config.example.json` to `backend/config.json`.

## Date Formats

- Full dates: `dd-mm-YYYY` (example: `05-03-2026`)
- Recurring holidays: `dd-mm` (example: `25-12`)

## Top-level Keys

### `agents` (required)
- Type: array of objects
- Purpose: define every agent and individual constraints.

Agent object fields:
- `name` (string, required): unique agent identifier.
- `preferences` (object, required):
  - `preferred` (array of shifts)
  - `avoid` (array of shifts)
- `restriction` (array of shifts, required): shifts permanently forbidden for this agent.
- `unavailable` (array of full dates, required): dates where the agent cannot work.
- `training` (array of full dates, required): dates blocked for training.
- `exclusion` (array of full dates, required): additional blocked dates.
- `vacations` (array of periods, required):
  - each item: `{ "start": "dd-mm-YYYY", "end": "dd-mm-YYYY" }`

Allowed shift names:
- `Jour`
- `Nuit`
- `CDP`

### `vacations` (required)
- Type: array of shift names
- Purpose: shifts generated by the solver each day.
- Typical value: `["Jour", "Nuit", "CDP"]`

### `vacation_durations` (required)
- Type: object
- Required keys:
  - `Jour` (number)
  - `Nuit` (number)
  - `CDP` (number)
  - `Conge` (number)
- Purpose: paid-hour durations used by balancing constraints.

### `holidays` (required)
- Type: array of `dd-mm`
- Purpose: recurring public holidays used by scheduling rules.

### `solver` (optional, recommended)
- Type: object
- Purpose: tune runtime/performance and balancing strictness.

Supported keys:
- `max_time_seconds` (integer, default `600`)
- `relative_gap_limit` (number in `(0, 1]`, default `0.1`)
- `num_search_workers` (integer, default `0`)
- `global_max_gap` (integer, default `240`)
- `period_max_gap` (integer, default `240`)
- `optimize_period_balance` (boolean, default `false`)
- `period_balance_weight` (integer, default `2`)

## Common Mistakes

- Using `YYYY-mm-dd` instead of `dd-mm-YYYY` in agent dates.
- Adding comments inside JSON (JSON does not support comments).
- Using unknown shift names (must match exactly `Jour`, `Nuit`, `CDP`).
- Forgetting required agent keys (all agent fields are expected, use empty arrays if needed).
- Setting solver keys with wrong types (for example `"0.1"` as a string).
