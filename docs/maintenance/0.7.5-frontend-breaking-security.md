# 0.7.5 Frontend Security Migration

## Objective
Resolve remaining frontend runtime vulnerabilities linked to the Jest/jsdom chain without impacting application runtime behavior.

## Baseline (before migration)
- Command: `npm audit --omit=dev`
- Result date: 2026-03-13
- Current status: 4 low-severity vulnerabilities
- Root cause chain: `@tootallnate/once` -> `http-proxy-agent` -> `jsdom` -> `jest-environment-jsdom`
- Initial remediation hypothesis: breaking upgrade (`npm audit fix --force` equivalent scope)

## Scope
- Frontend dependency hygiene only.
- Keep runtime dependencies limited to application runtime packages.
- Move test-only dependencies to `devDependencies` so runtime audit excludes internal test tooling.

## Out of Scope
- Backend dependency updates (unless a hard compatibility blocker appears).
- Functional product features.
- UI/UX changes unrelated to test/security migration.

## Acceptance Criteria
- `npm audit --omit=dev` reports no remaining findings from the current Jest/jsdom chain.
- `npm test -- --runInBand` passes.
- CI frontend test job remains green.
- Changelog entry added under `Unreleased` summarizing security remediation.

## Outcome
- Applied strategy: moved Jest and Vue test tooling packages from `dependencies` to `devDependencies`.
- Result: `npm audit --omit=dev` now reports `found 0 vulnerabilities`.
- Validation: frontend test suite still passes.
