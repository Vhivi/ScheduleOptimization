# 0.7.5 Frontend Breaking Security Migration

## Objective
Resolve remaining frontend runtime vulnerabilities that require breaking dependency upgrades in the Jest/jsdom test stack.

## Baseline (before migration)
- Command: `npm audit --omit=dev`
- Result date: 2026-03-13
- Current status: 4 low-severity vulnerabilities
- Root cause chain: `@tootallnate/once` -> `http-proxy-agent` -> `jsdom` -> `jest-environment-jsdom`
- Required remediation path: breaking upgrade (`npm audit fix --force` equivalent scope)

## Scope
- Frontend dependencies and test tooling only.
- Upgrade and align Jest ecosystem packages (including `jest-environment-jsdom`) to eliminate current runtime audit findings.
- Update test/config files as needed for compatibility.

## Out of Scope
- Backend dependency updates (unless a hard compatibility blocker appears).
- Functional product features.
- UI/UX changes unrelated to test/security migration.

## Acceptance Criteria
- `npm audit --omit=dev` reports no remaining findings from the current Jest/jsdom chain.
- `npm test -- --runInBand` passes.
- CI frontend test job remains green.
- Changelog entry added under `Unreleased` summarizing breaking tooling migration.
